## [1.2.1] - 2026-06-13

v1.2 ラインのフォローアップリリース。字幕編集まわりの小さな追加
(行の複製、 改行 2 系統、 プレビューでの複数字幕の重ね表示) と、
v1.2.0 後に出たいくつかの不具合修正。

### 新機能

- **字幕の行を複製できるようになりました。** 一覧とタイムラインの
  ブロックインスペクター、 どちらにも複製ボタンを追加。 複製した行は
  一覧では元の行のすぐ下に挿入され、 テキスト・時刻・フォント・色・
  アウトライン・フェードといったすべてのスタイル属性をそのままコピー
  します。 複製した行は「編集済み」扱いになるので、 後から見ても
  どの行が複製かが一目で分かります。
- **改行ボタンが 2 種類** になりました (旧「自動改行」を置き換え):
  - **敷き詰め改行** — 既存の改行をいったん消して、 画面の幅
    いっぱいに詰め直します。 旧「自動改行」と同じ挙動。
  - **はみ出し改行** — 手動で入れた改行はそのままに、 画面から
    はみ出す部分だけを折り返します。
  両ボタンは一覧の行末アイコン、 一括編集バー、 タイムラインの
  ブロックインスペクターの 3 か所に追加。
- **プレビューで複数字幕を重ねて表示。** 同じ時刻に 2 つ以上の
  字幕が重なる場合、 書き出し後の動画と同じ縦並び順 (先頭の字幕が
  端、 後続が端から離れる方向) でプレビューに表示されるようになり
  ました。 これまでは 1 つしか表示されていませんでした。

### 改善

- 旧「自動改行」を **「敷き詰め改行」** に名称変更しました。
  新しい「はみ出し改行」と並べて区別しやすくするための整理で、
  既存ボタンの挙動は変えていません。

### 修正

- テキスト入力中に改行ボタンを押すと、 整形結果がフォーカスを
  外した瞬間に元に戻ってしまうことがある問題を修正しました
  (直前に打ったキーが失われる場合もありました)。 入力フィールドは、
  ユーザーが手を入れていない間は外部からの更新を取り込み、
  実際に手で入力した場合だけフォーカスを外したときに確定する
  仕組みに改めました。
- タイムラインで同じ開始時刻のクリップをドラッグすると、 別の
  クリップが動いてしまう問題を修正しました。 ドラッグ中はクリップの
  表示位置 (行) がドラッグ開始時点で固定されるようになりました。
- プレビューで複数字幕を表示しているとき、 早く終わった字幕が
  消えると残りの字幕が下に詰めて表示される問題を修正しました。
  書き出し (libass) は各字幕の縦位置を「登場した時点」で確定して
  以降固定する仕組みなので、 プレビューもそれに合わせて、 一度
  決まった位置は動かないようにしました。 後から登場する字幕は
  空いた場所に入ります。

## [1.2.0] - 2026-06-10

第 3 マイナーラインのリリース。タイムライン編集 (非破壊なトリミング)、
はさみマーカーによる取り消し、タイムラインの使い方ガイド、
エラー / 警告分類の再整理。

### 新機能

#### タイムライン編集 — トリミング

- STEP 2 のタイムラインビューに新しい **トリミング** 操作を追加。
  タイムラインのルーラー上で始点と終点を打ち、 **トリミング** を押すと、
  動画の尺ごとその区間が削除され、 後続の字幕が前に詰められる。
  操作は非破壊で、 元の字幕データは保持されたまま、 表示上の時間軸だけが
  縮む。
- **はさみマーカー** が各トリミング位置に残り、 クリックでそのトリミングを
  取り消して該当区間の字幕を復元できる。 トリミングが入れ子になった場合は
  外側から順に取り消せる (内側は外側を取り消した次の描画から
  クリック可能になる)。
- トリミングされた区間に完全に含まれる字幕は一時的に非表示
  (テーブルでは「トリミング削除」と表示)、 そのトリミングを取り消すと
  自動で復活する。
- タイムラインツールバーに「使い方」ポップを設置。 トリミング操作 /
  はさみマーカーで戻す / ズーム・吸着 / 字幕を一行に収める、 の 4 つの
  ヒントを表示。

#### 問題ありタブとエラー / 警告の分類

- 「警告」タブを **「問題あり」** に改名。 ユーザーが見るべき問題を
  一画面に集約。
- 問題は次の 2 段階に分類:
  - **エラー** (赤バッジ): 書き出しを阻止 — 時間不正、 動画の尺超過、
    サイズ不正。 エラーが残っている間は「書き出しへ」ボタンを無効化し、
    tooltip で「問題ありタブで赤バッジを修正してください」と案内。
  - **警告** (アンバー): 書き出しは可能だが要確認 — テキスト未入力、
    時間重複、 はみ出し。
- 状態列のバッジも赤 (エラー) / アンバー (警告) に色分け。

### 改善

- **時刻表示を編集後の時間軸 (Edited 軸) に統一**: STEP 2 のすべての
  時刻表示 (テーブルセル / 時刻調整ダイアログ / タイムラインルーラー /
  インスペクター) を、 トリミング適用後の出力時刻に統一。 以前は
  テーブルが元の時間軸を表示していたためタイムラインと噛み合っていなかった。
- **削除クリップを凍結**: トリミングで削除された字幕はテーブル上で
  薄く表示され、 編集操作 (テキスト / 時間 / サイズ / スタイル / 一括 /
  削除 / 復元 / 自動改行 / フォント / 色 / フェード) を受け付けなくなる。
  復元するには該当区間のはさみマーカーをクリックする必要がある。
- 「カット」を **「トリミング」** に統一 (ボタンラベル、 tooltip、
  はさみマーカーのタイトル、 トースト、 使い方ガイド本文すべて)。
- トリミングツールバーを **始点 → 終点 → トリミング** の操作順に配置、
  「トリミング」見出しラベルは削除 (枠で囲んでいるので不要)。
- トリミングされた区間に含まれる行に対する一括編集 / リセットは
  スキップ (= 凍結行が予期せず編集されない)。

### 修正

- 既にトリミングされた字幕の中間に追加のトリミングを行う際、 2 つの
  トリミングを正しく union 化 (以前は二重トリミングで字幕が過剰に
  縮められる場合があった)。
- 完全に他のトリミングに含まれる内側のはさみマーカーを非表示に
  (= クリック可能なのは最外殻のみ)。
- 「問題あり」タブの件数を union 集合で正しく計上 (重複カウントを除去)。
- 「削除済み」タブの行アクションボタン (復元等) が見えてクリック
  できるように。
- 使い方ポップを右方向 (= 使い方ボタンの右隣) に開くように変更。
  これまでは下方向に開いていたため、 翻訳の長い英語環境などで下端が
  見切れていたが、 横方向に開くことで viewport の縦制約から解放。

### テスト・インフラ

- E2E テストのセレクタを `button:has-text("追加")` 等のローカライズ依存
  から `data-testid` に変更。 `DEFAULT_LANGUAGE` がどちらでも green
  になる。

## [1.1.1] - 2026-06-02

第 2 マイナーラインのリリース。行ごとフォント指定、音声ファイル入力対応、
カラーパレット拡張、設定ダイアログの再構成。

### 新機能

#### フォントカスタマイズ
- 9 フォントのレジストリ: Noto Sans JP SemiBold (同梱) + 8 種の
  Google Fonts (Dela Gothic One / Reggae One / Yusei Magic / Mochiy Pop One /
  Hachi Maru Pop / Potta One / DotGothic16 / Rampart One)
- 「選択 + 管理」を 1 リストに統合したフォントピッカーを字幕スタイル
  ダイアログと設定 ▸ フォントタブで共有
  - 行クリックでプロジェクト既定フォントを設定、各行にダウンロード /
    削除アイコン
  - 各フォントのライセンス (SIL OFL v1.1) を upstream verbatim の OFL.txt で表示
- STEP 2 字幕テーブルで行ごとにフォントを上書き可能
  - 各行のテキスト編集欄の上にコンパクトなフォントセレクタ
  - 一括編集バーにも同等のフォントピッカーを追加し、複数行に一括適用
  - 行ごとのフォントは ASS `\fn<family>` と staged fontsdir 経由で焼き込みに
    流れるので、1 本の出力動画に複数フォントが混在可能
- Hachi Maru Pop / Potta One に「稀な漢字非対応」警告 (塡 / 剝 / 頰 等
  常用漢字の一部を欠くため)
- 焼き込み前のフォント検証: 未ダウンロードのフォントを参照する行があれば
  保存ダイアログを開く前にトーストで通知 (バックエンドの失敗より前で弾く)
- 行で使用中のフォントを削除する際は確認ダイアログ表示。削除確定時には
  対象行の `fontId` を自動クリアして通知トースト

#### 音声ファイル入力
- 動画形式に加えて、6 種類の音声形式に対応: MP3 / WAV / M4A / AAC (raw ADTS)
  / FLAC / OGG (Vorbis)
- ffprobe による中身ベースのモード判定 — 拡張子偽装 (例: 動画を `.mp4`
  に維持したまま `.mp3` にリネーム) は無視され、実ストリーム構成で判定
- STEP 1 の音声モード UI: サムネイル枠の代わりに音声波形アイコン、
  解像度行非表示、フォーマット行はコンテナ + 音声コーデックの簡潔表示
- STEP 2 の音声モード UI: AudioPreviewPanel (再生 / 一時停止 + シークバー
  + 時刻表示) が VideoPreviewPanel と差替、各行の サイズ / スタイル /
  フォント セル非表示、一括編集バー非表示、「書き出しへ」非表示、
  テキスト / SRT 出力が唯一の出力経路に
- 音声再生中、再生位置に応じて字幕テーブルの行フォーカスが追従
  (動画モードと同じ挙動)

#### カラーパレット
- 30 色を 3 グループに分けて全ピッカー (行ごと / 一括 / 設定 /
  字幕スタイル) で統一表示:
  - 基本色 10 色
  - おすすめの組み合わせ 5 ペア (文字色 × アウトライン色を 1 クリックで同時セット。
    タイル上には字幕風の「Aa」プレビューを描画)
  - 色弱向け 10 色 (Color Universal Design 推奨配色)
- popover の右上に × 閉じるボタン、Radix の `--radix-popover-content-
  available-height` に追従するため設定ダイアログ内で開いても見切れない

#### 設定ダイアログの再構成
- 4 タブ構成に再構成: 一般 / フォント / 既定スタイル / Whisper設定
- アンダーライン式のタブ表示 (旧チップ式はダイアログと同色のため
  非選択タブが平文に見えていた)
- フォント管理はフォントタブ、既定スタイルと Whisper エンジン設定は
  それぞれ専用タブを持ち、STEP 1 の対応サーフェスと同じ store スライスを共有

### 修正

- **Whisper モデル DL 整合性チェック (C-3)** — ダウンロード完了後にハッシュ
  検証を行ってからインストール済みとマーク
- **APP_VERSION の動的化** — ウィンドウタイトル / About ダイアログ / 起動
  ログが `package.json` の version をビルド時に取得 (バージョン bump 時の
  並行編集が不要に)
- **OFL 順守** — 全フォント (同梱の Noto を含む) が `google/fonts/ofl/
  <name>/OFL.txt` から **verbatim** で取得した個別 OFL.txt を同梱。以前の
  per-font OFL の編集 / 合成パスは撤去し、upstream の原本のまま配布する
  方針に統一
- **設定ダイアログ** — 開いた瞬間に active な tabpanel に緑のフォーカス
  リングが出ていた問題を解消 (キーボード操作時のみリング表示)。4 タブ
  すべて同じ高さで安定するので切り替え時にジッタが出ない
- 音声入力時の言い回しを中立化 (「動画を読み込みました」「動画の読み込みに
  失敗しました」等 → 「ファイルを読み込みました」「ファイルの読み込みに
  失敗しました」)
- その他多数の UI 微調整: 行ごとフォントセレクタとテキスト欄の幅揃え、
  フォント行を単一インジケータドットに簡素化、「ダウンロード」完了時の
  自動選択を撤去 (DL とアクティブフォント変更を分離)、フォントサイズの
  上限を tooltip + インライン補足で可視化 (旧来の silent 200px clamp が
  ユーザに見える形に)

## [1.1.0] - 2026-06-01

- Step 2：複数行選択（チェックボックス列を追加）

- Step 2：一括編集バー — 文字色 / アウトライン色 / アウトライン太さ /
  フェードを選択中の複数行に一括適用

- Step 2：選択行への自動改行一括実行 — グリフ単位の正確な幅計算で
  動画幅に合わせて再改行
- Step 1：字幕スタイルダイアログにライブサンプルプレビューを追加
- Step 1：詳細な文字起こし設定（VAD 閾値、ビームサイズ、言語等）を
  専用ダイアログに分離
- アウトライン太さスライダを Step 1 スタイルダイアログ / Step 2 行単位 /
  Step 2 一括編集の 3 箇所で統一

### 変更

- デフォルトウィンドウ 1280×820 に各ステップの初期表示を収めるための
  UI 全面再構成：
  - Step 1：動画 + 音声トラックを中心としたコンパクト表示。

  最小 960×640
- 「自動改行」設定を「Advanced」項目から字幕スタイル既定値に移動。
  プレビューにリアルタイム反映
- 新規 UI コードの色指定を shadcn テーマトークン経由に統一
  （将来のライトテーマ対応のための準備作業）

### 修正

- コマンドパレットの「プロジェクトを支援する」項目から寄付ダイアログが
  開かなかった問題を修正

## [1.0.0] - 2026-05-30

初回パブリックリリース。

### 新機能

#### 動画入力
- 動画ファイル選択（MKV、MP4、MOV、AVI 対応）
- 動画メタ情報表示（解像度、長さ、フォーマット、ファイルサイズ）
- サムネイルプレビュー
- カスタムプロトコル `mojioko-media://` によるアプリ内動画再生（HTTP Range 対応）

#### 音声トラック選択
- マルチトラック音声の列挙（コーデック、チャンネル、サンプリングレート表示）
- トラック選択 UI、デフォルトトラック自動選択
- OBS マルチトラック録画向けに最適化

#### Whisper モデル管理
- 3 種類のモデル対応：`small`、`medium`、`large-v3`（Systran faster-whisper int8 量子化版）
- HuggingFace からのモデルダウンロード（ファイル単位の進捗表示）
- ダウンロード前のディスク容量チェック
- キャンセル可能なダウンロード（部分ファイルの自動クリーンアップ）
- アクティブモデルの切替、アンインストール、フォルダアクセス

#### 文字起こし
- faster-whisper Python sidecar（Python ランタイム不要、PyInstaller でバンドル）
- CPU + int8 量子化（GPU 不要、Windows PC で動作）
- VAD（音声検出）：閾値・継続時間の調整可能
- ビームサイズ調整（1-20）
- 11 言語対応（自動検出、日本語、英語、中国語、韓国語、スペイン語、フランス語、ドイツ語、ポルトガル語、ロシア語、アラビア語）
- リアルタイム進捗（1% 単位）
- 確認ダイアログ付きキャンセル機能
- 動画幅に合わせた自動改行（オプション）

#### 字幕エディタ（Step 2）
- テーブルビュー：# / 時間 / サイズ / スタイル / テキスト / 状態 / 操作 列
- 5 タブフィルタ：すべて / 出力対象 / 編集済み / 警告あり / 削除済み
- インラインテキスト編集（Ctrl+Enter / blur でコミット）
- インライン時間編集（HH:MM:SS.cc 形式）
- 時間調整ダイアログ：±1 秒 / ±0.1 秒 ステッパー、長押しリピート、前後行スナップ
- 行追加（開始時刻に応じた自動挿入）
- ソフト削除と復元
- 元の状態にリセット（テキスト / 時間 / スタイル）
- Undo / Redo（最大 100 ステップ）
- 時間編集後の自動ソート
- フォーカス行への自動スクロール（framer-motion アニメ完了後）
- テキスト書き出し（`.txt`、`\N` 改行除去）
- SRT 書き出し（UTF-8 BOM、DaVinci Resolve 互換）
- 動画プレビューパネル：再生・シーク、再生時刻と字幕の自動連動

#### 字幕スタイル設定
- フォントサイズ：30-200 px
- 文字色（HEX カラーピッカー、セッション履歴付き）
- アウトライン色（同上）
- アウトライン太さ：0-10 px
- フェード on/off
- フェード時間：0.1-0.5 秒（アプリ全体の設定）

#### 警告・出力可否判定
- **出力対象** = `!isDeleted && !emptyText`
- **焼き込み対象** = `isOutputTarget && !timeInvalid && !overDuration && !invalidSize`
- ステータスバッジ表示：時間不正、時間超過、時間重複、サイズ不正、はみ出し、テキスト未入力、削除済み
- はみ出し箇所の赤色強調表示

#### 動画焼き込み（Step 3）

### 既知の制限事項

- **コード署名なし**：初回起動時に SmartScreen 警告が表示される場合があります

- **macOS / Linux**：v1.0.0 では未対応（Windows のみ）
- **同梱フォント 1 種類のみ**：Noto Sans JP SemiBold（フォント切り替え機能は今後実装予定）
- **文字色・アウトライン色の不透明度なし**：100% 不透明のみ（透過値設定は今後実装予定）
